## How to Contribute

### 1. Report Issues
- Use [GitHub Issues](https://github.com/HAuser1234/homeassistant-local-weather-forecast/issues)
- Include:
  - Home Assistant version
  - Integration version
  - Detailed description
  - Steps to reproduce
  - Relevant logs

### 2. Suggest Features
- Open a [Feature Request](https://github.com/HAuser1234/homeassistant-local-weather-forecast/issues/new)
- Describe the use case
- Explain expected behavior

### 3. Submit Code Changes

#### Quick Process:
1. **Fork** the repository
2. **Create** a feature branch: `git checkout -b feature/my-improvement`
3. **Make** your changes
4. **Test** thoroughly (see TESTING.md)
5. **Commit** with clear messages
6. **Push** to your fork
7. **Create** a Pull Request

#### Code Standards:
- Follow Home Assistant coding standards
- Use type hints
- Add comments for complex logic
- Update documentation if needed
- Test with real hardware if possible

#### Testing:
- Run `pytest` if available
- Test manually in Home Assistant
- Check for errors in logs
- Verify existing functionality still works

### 4. Improve Documentation
- Fix typos
- Add examples
- Translate to new languages
- Update guides

### 5. Add Translations
See `custom_components/local_weather_forecast/translations/` for examples.

Required files:
- `translations/XX.json` (where XX is language code)
- Update `const.py` with new language
